üîë Token Management

Generating Access Tokens

1. Dashboard Admin Token 

# Generate a token for the admin-user
kubectl -n kubernetes-dashboard create token admin-user

# The token will be displayed in the output

2. Service Account Token 

# Generate a token for a specific service account
kubectl -n <namespace> create token <service-account-name>

Token Types

  1. Dashboard Access Token
  2. Used for accessing the Kubernetes Dashboard
  3. Generated for the admin-user service account

  4. Provides cluster-wide access

  5. Service Account Token

  6. Used for service-to-service authentication
  7. Scoped to specific namespaces and permissions

  8. Generated for individual service accounts

  9. Port Forward Token

  10. Used by the divine-port-forward service
  11. Provides access to port forwarding capabilities
  12. Scoped to specific services

Token Management

  1. Token Rotation
  2. Regularly rotate service account tokens
  3. Use short-lived tokens for temporary access

  4. Implement token expiration policies

  5. Access Control

  6. Follow the principle of least privilege
  7. Use namespaced service accounts when possible

  8. Implement RBAC rules for fine-grained control

  9. Token Storage

  10. Never store tokens in code or version control
  11. Use Kubernetes secrets for token storage
  12. Implement secure token distribution methods

RBAC Configuration 

# Example RBAC configuration for dashboard access
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: omega-dashboard-admin
subjects:
- kind: ServiceAccount
  name: admin-user
  namespace: kubernetes-dashboard
roleRef:
  kind: ClusterRole
  name: cluster-admin
  apiGroup: rbac.authorization.k8s.io
